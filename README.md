Fraud / Anomaly Monitoring &
Investigation Dashboard
Background
I  am a Business Analytics Associate at an e-commerce company. Finance and
Operations teams report a rise in:
suspicious transactions (promo abuse, card testing, fake COD orders)
high return/refund losses
unusually high discount usage and repeat cancellations
Leadership wants an always-on fraud/anomaly monitoring system that answers:
1. What are the biggest fraud/anomaly patterns right now?
2. Which customers/sessions/orders should be investigated first (risk ranking)?
3. What is the estimated loss and how can we reduce it?
4. What experiments / controls should be introduced and how will success be
measured?
You will build an end-to-end solution: clean data layer, anomaly rules + scoring, weekly
trends, investigation queue, and a dashboard.
This is an analytics + monitoring capstone (not “hard fraud ML”). Students can optionally
build an ML anomaly model, but rule-based + explainable scoring is mandatory.
